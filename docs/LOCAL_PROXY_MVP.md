# 本地 AI 代理使用指南（给普通用户）

## 这是什么

错题库的 AI 功能（图片识题、重新解题）需要调用大模型 API。但有些模型服务商不允许浏览器跨域请求，导致网页无法直接调用。

这个本地代理就是你电脑上的一个「中转站」：网页请求发到本机 127.0.0.1，代理再转发给模型服务商，回来时加上正确的 CORS 响应头。

**127.0.0.1 就是你自己的电脑**。每个人都要在自己的电脑上运行这个代理。

## 为什么每个人都要自己运行

- 网页里的 `127.0.0.1` 指的是**当前这个浏览器的电脑**，不是服务器。
- API Key 只在你自己电脑的浏览器和代理之间传递，**不会上传到错题库服务器**。
- 代理只监听 `127.0.0.1`，局域网其他电脑和公网都无法访问。

## 准备条件

- 一台 Windows 电脑
- [Node.js LTS](https://nodejs.org/)（安装后需重启或重新打开终端）
- 一个可用的大模型 API Key 和 Base URL（如智谱 BigModel、OpenAI 等）
- 确认你的模型支持所需功能（文字识题只需要文本模型，图片识题需要视觉模型）

## 使用步骤

### 1. 获取代理包

管理员会给你一个 `local-llm-proxy.zip`，解压到任意目录（建议不要有中文路径）。

### 2. 启动代理

双击 `start.bat`。

第一次运行会自动：
- 提示你编辑 `.env`（只有 PORT 和 ALLOWED_ORIGINS，不需要在这里填 API Key）
- 安装依赖（需要网络）

编辑完成后关闭记事本，**再次双击 `start.bat`**，看到以下输出表示成功：

```
[proxy] 本机 LLM 代理已启动
[proxy] 监听: http://127.0.0.1:8787/v1/chat/completions
[proxy] 允许 CORS Origins: http://8.148.71.66, http://localhost:3000, ...
```

**保持这个窗口打开，不要关闭。** 关闭窗口即停止代理。

### 3. 验证代理

浏览器打开 http://127.0.0.1:8787/health，应返回：

```json
{
  "ok": true,
  "service": "wrong-notebook-local-llm-proxy",
  "allowedOrigins": ["http://8.148.71.66", ...]
}
```

### 4. 配置错题库网页

打开 http://8.148.71.66，进入「设置」页：

**方法一：使用本机代理（推荐，解决 CORS）**

- 开启「使用本机代理解决 CORS」
- Proxy URL 填：`http://127.0.0.1:8787/v1`
- API Key 填你的模型 API Key
- Base URL 填你的模型服务商地址
- Model 填你的模型名
- 点击「检测本机代理」→ 通过
- 点击「测试连接」→ 通过

**方法二：直连（仅当模型服务商已支持 CORS）**

- 关闭「使用本机代理解决 CORS」
- API Key、Base URL、Model 同上
- 点击「测试连接」→ 通过

### 5. 使用 AI 功能

配置完成后即可使用：
- 首页「文字 AI 解题」
- 首页「拍照识题」（需视觉模型）
- 错题本「重新解题」

## 示例配置

### 智谱 BigModel

| 字段 | 值 |
|------|-----|
| Base URL | `https://open.bigmodel.cn/api/paas/v4` |
| API Key | 你的 BigModel API Key |
| Model | `glm-4v-plus`（视觉）或 `glm-4-flash`（纯文本） |

### OpenAI 兼容接口

| 字段 | 值 |
|------|-----|
| Base URL | `https://api.openai.com/v1` 或兼容地址 |
| API Key | `sk-xxxx` |
| Model | 按你的账号实际可用模型填写 |

> 不同服务商的模型名和 Base URL 不同，请以你的服务商文档为准。

## 常见错误

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| /health 打不开 | 代理没启动 | 双击 start.bat 并保持窗口打开 |
| /health 打不开 | 端口被占用 | 改 .env 的 PORT，网页端 Base URL 也要同步改 |
| 网页检测失败 | 代理没启动 | 确认 start.bat 窗口没关闭 |
| 网页检测失败 | CORS Origin 不匹配 | 检查 .env ALLOWED_ORIGINS 是否包含当前网页地址 |
| 测试连接 401 | API Key 错误 | 检查网页设置里填的 API Key |
| 测试连接 404 | Base URL 或模型接口路径错误 | 确认 Base URL 正确（通常以 /v1 结尾） |
| 模型不存在 | 模型名错 | 确认模型名与你的服务商支持的一致 |
| 返回空内容 | 模型不兼容 | 某些 thinking 模型 content 可能为空，换模型试试 |
| 图片识题失败 | 模型不支持视觉 | 换用支持 image_url 的视觉模型 |
| 端口冲突 | 8787 已被占用 | 改 .env 的 PORT 为其他值（如 8788），网页端同步改 |

## 安全说明

- **不要把 .env 发给别人**（如果里面有你的配置）
- **不要把 API Key 截图分享**
- 用完可以关闭代理窗口
- 公共电脑用完记得删除 .env
- **不要把 HOST 改成 0.0.0.0**，这会让局域网其他设备也能访问
- 不要在陌生网站填写 API Key

## 停止代理

关闭 start.bat 打开的窗口，或按 Ctrl+C。
